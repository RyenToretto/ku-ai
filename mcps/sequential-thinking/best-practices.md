# Sequential Thinking MCP 最佳实践

## 推荐做法

- 面对复杂问题时主动使用，比一次性输出长答案更可靠
- 每步思考聚焦于一个子问题
- 如果发现之前的步骤有误，使用修正功能而非重新开始
- 利用分支探索不同解决方案

## 适用场景

- 需要严谨推理的技术方案设计
- 排查多层嵌套的 bug
- 评估多个架构方案的优劣
- 需要回溯和修正的复杂分析

## 不适用场景

- 简单直接的代码修改
- 已有明确方案的执行类任务
- 纯文件操作

## 与其他工具配合

- **Systematic Debugging Skill**：调试复杂 bug 时，先用 sequential-thinking 做根因分析
- **Brainstorming Skill**：思维链可作为头脑风暴的结构化补充

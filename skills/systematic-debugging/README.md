# Skill: systematic-debugging

> 系统化调试流程，遇到 bug/测试失败/异常行为时在修复前先做充分分析。

## 基本信息

| 项目 | 值 |
|------|-----|
| Skill 名称 | `systematic-debugging` |
| 来源 | `superpowers` 插件 / 全局预装 |
| 安装位置 | `~/.agents/skills/systematic-debugging/` |
| 触发条件 | 遇到任何 bug、测试失败或异常行为时 |
| 依赖 | 无 |

## 核心流程

1. **理解问题** — 复现问题，明确预期 vs 实际行为
2. **收集证据** — 日志、错误信息、堆栈追踪、网络请求
3. **形成假说** — 基于证据列出可能的原因
4. **验证假说** — 逐一测试，缩小范围
5. **修复** — 确认根因后再修复
6. **验证修复** — 确保修复有效且未引入新问题

## 核心原则

- **先理解再修复**：不要急于猜测和尝试
- **证据驱动**：每个假说都需要证据支持
- **不要假设**：代码的实际行为可能和你想的不一样
- **一次改一处**：便于确认哪个修改解决了问题

## 与其他工具配合

- **Sequential Thinking MCP**：复杂 bug 用思维链做分步推理
- **Chrome DevTools MCP**：在运行时收集证据
- **Playwright MCP**：自动化复现步骤

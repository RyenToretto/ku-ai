# code-quality.mdc 设计说明

## 设计意图

建立所有项目共享的代码质量底线，确保 AI Agent 生成的代码符合基本工程规范。

## 适用场景

- 所有编程语言和框架
- 自动应用（`alwaysApply: true`）

## 核心要点

1. **DRY 原则** — 重复逻辑不超过两处
2. **函数约束** — 不超过 40 行、参数不超过 4 个、嵌套不超过 3 层
3. **命名规范** — camelCase 变量、UPPER_SNAKE 常量、is/has 布尔前缀
4. **类型安全** — 禁止 any、公共 API 显式标注返回类型
5. **注释哲学** — 只注释 "为什么"，不注释 "做了什么"

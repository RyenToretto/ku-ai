# dev-scenarios 规则说明

## 设计意图

在 AI 辅助开发过程中，很多操作步骤、踩坑经验、环境配置流程等有价值的过程信息会随着对话结束而丢失。此规则要求 AI 在完成「典型场景」操作后，自动将完整的命令日志输出为 Markdown 文件，归档到项目的 `docs/dev-scenarios/` 目录中。

这些日志可以作为：
- 新人入职的操作手册
- 团队的知识库和经验沉淀
- CI/CD 配置和环境搭建的参考
- 踩坑记录和解决方案的备忘

## 适用场景

- 所有项目（全局规则）
- 适用于任何涉及多步骤操作、环境配置、首次操作、踩坑修复等有团队复用价值的场景

## 判断标准

「典型场景」的判断标准：
1. 首次操作（项目中第一次执行某类操作）
2. 踩坑与修复（遇到非显而易见的错误并解决）
3. 多步骤流程（3 个以上步骤的完整工作流）
4. 环境配置（开发环境、CI/CD、工具链）
5. 破坏性变更（major 升级、架构调整）
6. 团队复用价值（其他成员可能遇到同样问题）

## 输出位置

`<项目根目录>/docs/dev-scenarios/`

## 来源

全局 Cursor Rule：`~/.cursor/rules/dev-scenarios.mdc`
